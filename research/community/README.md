# Community Research

**สถานะ:** Draft
**ดำเนินการโดย:** Scout Agent (รันผ่าน Claude Cowork — ดู `SCOUT_COWORK_BRIEF.md`)

## ขอบเขต

เก็บ Evidence จากคอมมูนิตี้คนขับสาธารณะ (Facebook groups, Pantip, forum อื่นๆ) ตามหลัก **"Community = Evidence ไม่ใช่ Feature Request"** (`research/README.md`)

## Output ที่คาดหวัง (Scout v2.0)

- ไฟล์ Decision Case แยกตามเหตุการณ์ ตั้งชื่อแบบ `kebab-case` (ดู `docs/NAMING_CONVENTION.md`) เช่น `change-zone-rain.md`
- ทุกไฟล์ขึ้นต้นด้วย YAML metadata: `Platform`, `Decision`, `Situation`, `Tags`, `Evidence Status`, `Scout Confidence`
- เนื้อหาแบ่งเป็น 3 ส่วนเสมอ: **Decision Case** (Situation → Decision → Reason → Outcome), **Driver Vocabulary**, **Blind Spot**
- ทุกไฟล์ต้องขอ/ผูก **Evidence ID** ใหม่จาก `research/EVIDENCE_LOG.md` (ต่อจาก E-002 คือ E-003 เป็นต้นไป)
- ห้ามสรุปเป็น Feature idea — เก็บแค่ Decision Case/Evidence ตาม `research/RESEARCH_PRINCIPLES.md`
- ค้นหาจนถึง **Evidence Saturation** (ไม่พบ pattern ใหม่) ไม่ใช่หยุดตามจำนวนแหล่งตายตัว
