# Decision #018 — Daily Usage vs First Setup

**วันที่:** 2026-07-15
**สถานะ:** Proposed

## บริบท

การออกแบบเดิมปนกันระหว่างสิ่งที่ต้องทำ "ครั้งเดียวตอนตั้งค่า" กับสิ่งที่ต้องทำ "ทุกวัน" ทำให้ flow รายวันหนักเกินความจำเป็น

## การตัดสินใจ

แยกการออกแบบเป็น 2 ประเภทชัดเจน:

- **First Setup** — ทำครั้งเดียว (เช่น ข้อมูลรถ, ประเภทเชื้อเพลิง, ค่าเสื่อมเบื้องต้น)
- **Daily Usage** — ทำซ้ำทุกวัน ต้องเบาที่สุด (ดู Snapshot Flow, Decision #014)

## เหตุผล

Feature ที่ยอมให้ "หนัก" ได้คือ First Setup เพราะทำครั้งเดียว แต่ Daily Usage ต้องถูก optimize อย่างเข้มงวดตามหลัก Less Input, More Driving เพราะ friction ตรงนี้จะถูกคูณด้วยจำนวนกะที่คนขับทำ

## สิ่งที่เปลี่ยนแปลง

เมื่อเข้า Gate 3 (Wireframe) ให้แยก 2 flow นี้เป็นคนละ artifact ตั้งแต่ต้น ไม่ปนกันในเอกสารเดียว
