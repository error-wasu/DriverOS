# Decision #014 — Snapshot Flow

**วันที่:** 2026-07-15
**สถานะ:** Proposed

## บริบท

หลัง Decision #016 (GPS ไม่ใช่ Requirement) ต้องมีวิธีเก็บ Evidence ระยะทาง/เวลาแบบไม่พึ่ง GPS tracking ต่อเนื่อง

## การตัดสินใจ

ใช้ Daily Driver Flow แบบถ่ายภาพ (Snapshot) เลขไมล์ ต้น-ปลายกะ แทนการ track ตำแหน่งตลอดเวลา

```
เริ่มกะ:  Open App → Welcome → เมื่อคืนกำไร → Start Driving → Camera → ถ่ายเลขไมล์ → Done
จบกะ:     End → Camera → เลขไมล์ → Screenshot (รายได้สรุปจากแอปขับขี่) → Done
```

App ถูกเปิดแบบเต็มรูปเพียง 2 ครั้งต่อกะ (ต้น/ปลาย) — สอดคล้องกับ insight เดิมเรื่อง Notification-first ใน Gate 2 (ไม่ได้ขัดแย้งกัน เพราะ 2 ครั้งนี้คือช่วงที่คนขับ "ตั้งใจ" จะเปิดแอปอยู่แล้ว)

## เหตุผล

- ลด Input ให้เหลือน้อยที่สุด (แค่ถ่ายรูป ไม่ต้องพิมพ์)
- ไม่ต้องพึ่ง GPS/background tracking ที่ซับซ้อนและกินแบตเตอรี่
- Evidence (เลขไมล์) เพียงพอสำหรับคำนวณระยะทาง/ต้นทุนแบบคร่าวๆ

## สิ่งที่เปลี่ยนแปลง

- Journey เดิมใน Gate 2 ที่เน้น "ไม่ต้องเปิดแอประหว่างกะ" ยังคงอยู่ — เปลี่ยนเฉพาะช่วงต้น/ปลายกะที่ตอนนี้ระบุ flow ชัดเจนขึ้น
- ต้อง update Wireframe (Gate 3) ให้สะท้อน flow นี้เมื่อเริ่มทำ
