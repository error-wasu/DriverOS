# DriverOS Business Glossary

**สถานะ:** Draft
**Priority:** 🔴 Critical

## 1. Core Definitions

- **Session (กะ/เซสชัน):** ช่วงเวลาการทำงานเชิงธุรกิจของคนขับ เริ่มนับเมื่อกด [เริ่มวิ่ง] และสิ้นสุดเมื่อกด [เลิกวิ่ง]
- **Trip (เที่ยว/ทริป):** หนึ่งงานที่รับและส่งเสร็จสิ้น — เป็นหน่วยย่อยที่อยู่**ภายใน** Session หนึ่งกะมักมีหลาย Trip ไม่ใช่คำที่แข่งกับ Session แต่เป็นความสัมพันธ์แบบ Session ⊃ Trip (ดูหมายเหตุด้านล่าง)
- **Idle Time (เวลารอรับงาน):** ช่วงเวลาที่เปิดระบบสแตนด์บายแต่ไม่มีงานเข้า ไม่นับรวมช่วงเวลาที่ตั้งใจปิดระบบเพื่อพัก
- **Decision Case (เรื่องราวการตัดสินใจ):** บันทึกพฤติกรรมมนุษย์และบริบทหน้างานใน 1 เหตุการณ์ เพื่อศึกษาแรงจูงใจและผลลัพธ์

> **หมายเหตุจาก Scribe (sweep note):** ตรวจสอบแล้วว่า "Trip" และ "Session" ที่ปรากฏในเอกสารก่อนหน้า (เช่น Gate 2 Journey Map, `decisions/rejected/trip-tracking.md`) ไม่ได้ขัดแย้งกัน — ใช้ในความหมายลำดับชั้นที่สอดคล้องกันมาตลอด (Session = กะทั้งหมด, Trip = หนึ่งงานภายในกะ) จึงไม่ raise เป็น conflict แต่บันทึกนิยามอย่างเป็นทางการไว้ที่นี่กันความเข้าใจผิดในอนาคต

## 2. Evidence Status (สถานะของหลักฐานและความมั่นใจ)

- **Unknown:** สมมติฐานที่ยังไม่มีข้อมูลหรือข้อเท็จจริงหน้างานมารองรับ
- **Emerging:** เริ่มพบเจอพฤติกรรมนี้หลายครั้งในหน้างาน/คอมมูนิตี้ แต่ยังไม่สามารถยืนยันเป็นสถิติเชิงประจักษ์ได้
- **Validated:** มีหลักฐานเชิงพฤติกรรมหรือตัวเลขสถิติรองรับเพียงพอที่จะสรุปว่าเป็นความจริง

> **หมายเหตุ:** เอกสารก่อนหน้านี้ (เช่น `PROBLEM_LOG.md`, `CURRENT_BELIEFS.md`) เคยใช้คำว่า "🟢 Emerging Evidence" — ให้ถือว่าเทียบเท่ากับ **Emerging** ตามนิยามนี้ Scribe จะทยอย refactor คำให้ตรงกันทุกจุดเมื่อมีการแก้ไขเอกสารนั้นครั้งถัดไป (ตามกฎใน `docs/LANGUAGE_GUIDE.md`)
